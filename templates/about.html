⚙️ Stack technique

🔎 Découverte intelligente de liens

Avant de scraper, le système analyse les liens de la page principale et leur attribue un score de pertinence promo basé sur des mots-clés. Les sous-pages les mieux notées sont crawlées automatiquement — sans LLM, sans règles par restaurant.

  • Mots-clés positifs : promo, offre, commander, special…
  • Bonus +15 pour les routes /subcategory/, +5 pour les SPA #/
  • Filtrage des liens hors-domaine, légaux, login, réseaux sociaux
  • Jusqu'à 3 sous-pages crawlées par restaurant
Ex : bennyandco.ca/promotions → découvre automatiquement benny-co.com/commander/#/subcategory/7416
🕷 Scraping — Scrapling v0.4

Scrapling est un framework de scraping adaptatif avec trois modes de récupération, appliqués en cascade automatiquement selon le contenu de la page.

  • AsyncFetcher — HTTP rapide via curl_cffi (TLS fingerprinting, anti-bot stealth)
  • DynamicFetcher — Playwright/patchright (Chromium headless) pour les SPA JavaScript
  • StealthyFetcher — Camoufox (Firefox modifié) pour les sites avec détection avancée
  • Sélection automatique : HTTP d'abord, Playwright si contenu insuffisant (<1 500 chars)
from scrapling.fetchers import AsyncFetcher, DynamicFetcher
🧠 Extraction — GPT-4o-mini + GPT-4o

Le texte de chaque page est envoyé à un LLM avec un prompt structuré qui extrait toutes les promotions sous forme de tableau JSON.

  • gpt-4o-mini utilisé en priorité — 16x moins cher (~$0.15/M tokens)
  • gpt-4o utilisé en fallback (tentative 3) si mini retourne []
  • 3 tentatives : temperature=0 puis 0.3 pour varier le sampling
  • Retourne : promo_type, promo_details, price, promo_date, link, image_url
  • Multilingue FR/EN, sans règles par restaurant
openai.chat.completions.create(model="gpt-4o-mini")
🔍 Déduplication — exact match + embeddings optionnels

Avant d'insérer une promo, le système vérifie si elle existe déjà. Deux modes selon la configuration.

  • Mode rapide (SKIP_EMBEDDING_INIT=1) : exact text match sur promo_details
  • Mode sémantique : embedding vectoriel (modèle paraphrase-multilingual-MiniLM-L12-v2, 384 dim) + similarité cosinus
  • Seuil : 0.92 — si score ≥ seuil, met à jour last_seen au lieu d'insérer
🌐 Backend — Flask async + Hypercorn

L'application Flask utilise des threads isolés pour les scrapers. Chaque job possède son propre event loop asyncio, créé et détruit proprement.

  • Flask 3.x avec routes synchrones (scrape en thread séparé)
  • Hypercorn — serveur ASGI (remplace Gunicorn/uWSGI)
  • MySQL via mysql-connector-python
  • Frontend : Bootstrap 5 + JavaScript vanilla (fetch API, polling toutes les 2s)
  • APScheduler — scraping automatique toutes les {{ scrape_interval_hours }}h
⚡ Performance
  • Réponse immédiate (202) — aucun timeout HTTP côté client
  • Polling léger toutes les 2s via fetch('/job/<id>')
  • Scrapling AsyncFetcher : ~0.5–2s par page (HTTP)
  • DynamicFetcher (Playwright) : ~5–15s par page SPA
  • GPT-4o-mini : ~1–3s par extraction
  • Coût total estimé : ~$0.001–0.003 par restaurant/scrape

🏅 Système de grades

Chaque promotion est notée en fonction de son prix par rapport à une valeur de référence de 20 $ (repas moyen au restaurant). Les économies estimées = 20 $ − prix.

GradePrixÉconomies estiméesSignification
A+ < 7 $> 13 $ Excellente affaire — prix très bas
A 7 – 12 $8 – 13 $ Très bonne affaire
B 12 – 16 $4 – 8 $ Bonne affaire
C 16 – 20 $0 – 4 $ Légère réduction
D > 20 $0 $ Prix au-dessus de la référence
N/A Non fourni— Prix non extrait de la page

♻️ Gestion des promos expirées

✅
Active (is_active=1)

Promo vue lors du dernier scrape. Affichée normalement.

🔄
Confirmée

Promo déjà connue, re-vue lors du scrape → last_seen mis à jour.

⏸
Expirée (is_active=0)

Non vue depuis +{{ inactive_days }} jours → masquée par défaut, conservée en archive.

⚙️ Après chaque scrape, les promos non revues depuis {{ inactive_days }} jours sont automatiquement marquées comme expirées. Elles restent en base de données pour l'historique et peuvent être réactivées manuellement.

pip install -r requirements.txt
patchright install chromium   # navigateur pour les SPA JavaScript

# 2. Configurer l'environnement (.env)

DB_USER=root
DB_PASSWORD=1234
DB_NAME=promotions_db
OPENAI_API_KEY=sk-...
SKIP_EMBEDDING_INIT=1    # mettre à 0 pour activer la déduplication sémantique
SCRAPE_INTERVAL_HOURS=6  # fréquence du scheduler automatique

# 3. Lancer le dashboard
cd /home/as/ProjetUqacStage/app
bash run.sh

# 4. Accéder au dashboard
# → http://localhost:5000
