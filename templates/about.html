Architecture & ingénierie Architecture & engineering

Chicken Wings v2.0 — pipeline de scraping adaptatif, extraction IA, et système de nettoyage automatique. Chicken Wings v2.0 — adaptive scraping pipeline, AI extraction, and automatic deduplication system.

Pipeline d'Extraction Extraction Pipeline 4 STAGES
1

Découverte de liensLink Discovery

La page d'accueil de chaque restaurant est analysée pour trouver les sous-pages de promotions cachées (ex: /offres, /deals, /menu-specials). Un système de scoring par mots-clés priorise les liens les plus pertinents avant tout téléchargement. Each restaurant's home page is analyzed to discover hidden promo sub-pages (e.g., /deals, /offers, /menu-specials). A keyword scoring system ranks and prioritizes the most relevant links before any content download.

2

Scraping adaptatif (Scrapling)Adaptive Scraping (Scrapling)

Le moteur utilise la librairie Scrapling avec un pipeline à trois niveaux : Fetcher HTTP classique, puis curl_cffi avec impersonation TLS Chrome 131 pour contourner les WAF basiques, et enfin PlayWright via AsyncFetcher / StealthyFetcher pour les sites à rendu JavaScript complexe. Chaque niveau n'est déclenché que si le précédent échoue ou retourne un contenu insuffisant. The engine uses the Scrapling library with a three-tier pipeline: standard HTTP Fetcher, then curl_cffi with Chrome 131 TLS impersonation to bypass basic WAFs, and finally PlayWright via AsyncFetcher / StealthyFetcher for complex JS-rendered sites. Each tier is only triggered if the previous one fails or returns insufficient content.

3

Analyse par intelligence artificielleAI Analysis

Le HTML est converti en texte structuré (balises inutiles supprimées, marqueurs [IMAGE:url] préservés). Ce texte est envoyé à GPT-4o-mini (modèle primaire) ou GPT-4o (fallback) qui extrait noms, prix, types et dates sous format JSON. Pour les promotions visuelles (bannières), GPT-4o Vision est utilisé avec detail="low" (~85 tokens/image). HTML is converted to structured text (useless tags removed, [IMAGE:url] markers preserved). This text is sent to GPT-4o-mini (primary model) or GPT-4o (fallback) which extracts names, prices, types, and dates in JSON format. For visual promotions (banners), GPT-4o Vision is used with detail="low" (~85 tokens/image).

4

Nettoyage automatique post-scrapeAutomatic Post-Scrape Cleaning

Après chaque cycle, le système de nettoyage se déclenche automatiquement. Phase 1 : suppression des entrées non-promotionnelles via règles (textes de navigation, entrées trop courtes, sans prix ni mot-clé promo). Phase 2 : déduplication floue via SequenceMatcher (seuil 0.82, réduit à 0.68 si les prix coïncident). Peut aussi être déclenché manuellement depuis la page du restaurant. After each cycle, the cleaning system fires automatically. Phase 1: rule-based removal of non-promotional entries (navigation text, too-short entries, no price and no promo keyword). Phase 2: fuzzy deduplication via SequenceMatcher (threshold 0.82, lowered to 0.68 when prices match). Can also be triggered manually from the restaurant page.

curl_cffi — Chrome TLS Impersonation

Requête HTTP avec impersonate="chrome131". Bypasse la majorité des WAF de niveau 1 (Akamai, Cloudflare Bot Management basique). Rapide, sans overhead de navigateur. Détection du blocage Cloudflare via pattern matching sur le HTML retourné. HTTP request with impersonate="chrome131". Bypasses most tier-1 WAFs (Akamai, basic Cloudflare Bot Management). Fast, no browser overhead. Cloudflare block detection via HTML pattern matching on the response.

TIER 2
curl_cffi — Proxy Retry

Si un blocage IP Cloudflare WAF est détecté (hard ban), la même requête est relancée via le proxy défini dans SCRAPER_PROXY. Supporte les formats http://user:pass@host:port et socks5://host:port. Désactivé si la variable d'environnement est vide. If a Cloudflare WAF IP block is detected (hard ban), the same request is retried through the proxy defined in SCRAPER_PROXY. Supports http://user:pass@host:port and socks5://host:port formats. Disabled if the environment variable is empty.

TIER 3
Scrapling AsyncFetcher / StealthyFetcher (Playwright)

Navigateur Chromium headless piloté via Patchright (fork anti-détection de Playwright). Simule les comportements humains : mouvements de souris, délais aléatoires, fingerprint réaliste. Utilisé pour les sites à rendu JavaScript lourd (DoorDash, Tim Hortons) et les protections Cloudflare de niveau 2. Headless Chromium browser driven via Patchright (anti-detection fork of Playwright). Simulates human behaviors: mouse movements, random delays, realistic browser fingerprint. Used for heavy JS-rendered sites (DoorDash, Tim Hortons) and tier-2 Cloudflare protections.

L'application est entièrement conteneurisée. La base de données MySQL est isolée sur un réseau Docker interne inaccessible depuis l'extérieur. Le frontend communique avec Flask via une API asynchrone (polling toutes les 2 s) pour afficher les logs en temps réel sans bloquer l'interface. Le scraping automatique se déclenche toutes les {{ scrape_interval_hours }}h via APScheduler, suivi d'un nettoyage automatique. The application is fully containerized.
